# Voicebot Gateway (VBGW) 프로덕션 사용자 메뉴얼 (User Manual)

본 문서는 운영 중인 상용 Voicebot Gateway(VBGW)를 활용하여 **AI 대화 서비스를 구성하거나 연동하는 서비스 기획자 및 연동 개발자/사용자**를 위한 매뉴얼입니다.

---

## 1. VBGW 서비스 개요

VBGW는 사용자의 전화 음성(SIP/RTP)을 실시간으로 수신하여 AI 엔진(STT/TTS/LLM)으로 전달하고, 그 응답을 다시 통화 중인 고객에게 오디오로 들려주는 **"AI 전화 상담 중계기"**입니다. 사용자는 별도의 앱 설치 없이 전화를 거는 것만으로 AI와 대화할 수 있습니다.

### 주요 기능
- **초저지연 양방향 통화**: AI와 바로 대화하는 듯한 실시간 스트리밍 제공
- **말끊기 (Barge-in) 지원**: AI가 안내 멘트를 하는 도중 사용자가 말을 시작하면, 즉시 AI의 말을 끊고 사용자 말을 경청합니다.
- **다재다능한 IVR (자동응답) 및 단축키**: 전화 키패드(DTMF)를 활용해 메뉴 이동, 상담원 연결 등을 수행할 수 있습니다.

---

## 2. 콜봇 통화 연결 방법

### 2.1 인바운드 콜 (고객이 콜봇에게 전화 걸기)
고객이 PBX(대표 번호)를 통해 전화를 걸면 VBGW로 라우팅됩니다. 특별한 조치 없이 고객은 즉결된 음성 안내("안녕하세요, 인공지능 상담원입니다")를 수신하고 대화를 시작합니다.

### 2.2 아웃바운드 콜 (콜봇이 고객에게 전화 걸기)
회사의 서비스 서버에서 VBGW의 API를 호출하여 고객에게 전화를 걸 수 있습니다. 예약 안내, 알림용으로 쓰입니다.
**API 호출 예시:**
```json
POST http://{VBGW_IP}:8080/api/v1/calls
Body:
{
  "target_uri": "sip:01012345678@example-pbx.com"
}
```

## 3. 통화 중 제공되는 부가 기능 제어 (API)

진행 중인 통화 세션을 제어하고 싶을 때 사용하는 관리자 API입니다.
- **인증 방식**: 모든 요청 헤더에 `X-Admin-Key: {Admin 비밀키}` 포함

### 3.1 키패드(DTMF) 강제 전송
고객이 버튼을 누르는 대신 서비스 망단에서 강제로 이벤트를 주입할 수 있습니다.
- `POST /api/v1/calls/{call_id}/dtmf` (`{"digits": "1", "target": "ai"}`)

### 3.2 통화 브릿지 (상담원 연결)
사용자가 "상담원 연결해 줘"라고 요청하면 AI가 통화를 즉시 일반 상담원 콜(Session)과 결합시킵니다.
- `POST /api/v1/calls/bridge` (`{"call_a": 100, "call_b": 101}`)
- **가장 큰 이점**: 브릿지 모드 중에는 콜봇 AI 시스템이 잠들어(Pause), 상담원 통화 내용이 AI 서버로 유출되지 않으며 서버 리소스도 절약됩니다.

### 3.3 대화 녹음 제어
필요에 따라 각 통화의 Voice를 개별적으로 디스크(또는 연동된 스토리지)에 저장할 수 있습니다.
- 녹음 시작: `POST /api/v1/calls/{call_id}/record/start`
- 녹음 종료: `POST /api/v1/calls/{call_id}/record/stop`

## 4. 자주 묻는 질문 (FAQ)

**Q. 전화를 걸었는데 음성(TTS)이 들리지 않아요.**
A. 방화벽 문제이거나 망 내 코덱(G.711) 우선순위 문제입니다. 관리자에게 문의해 주세요.

**Q. 콜봇이 제 말을 중간에 자주 끊어요.**
A. 주변 소음으로 인해 말끊기(Barge-in)가 민감하게 동작했을 수 있습니다. 인프라 운영쪽에 `VAD 민감도` 하향 패치를 요청해 주세요.
